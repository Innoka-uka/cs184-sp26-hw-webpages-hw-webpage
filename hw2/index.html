Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm is a recursive method for evaluating Bezier curves. It works by linearly interpolating between consecutive control points to find intermediate points, repeatedly reducing the set of points until only one point remains. This final point lies on the curve at parameter \(t\).

I implemented this in the evaluateStep function. It takes a vector of control points and performs one level of interpolation. For each pair of consecutive points \(P_i\) and \(P_{i+1}\), it calculates a new point \(P'_i = (1-t)P_i + tP_{i+1}\). The function returns the vector of these new points. By calling this function repeatedly (as seen in drawCurve), the algorithm converges to the final point on the Bezier curve.

Part 2: Bezier surfaces with separable 1D de Casteljau

De Casteljau’s algorithm extends to Bézier surfaces by applying the 1D algorithm separably in two parametric directions (\(u\) and \(v\)).

For a Bézier surface defined by an \(n \times n\) grid of control points \(P_{i,j}\), I evaluate it in two stages:

  1. Fix u (row direction): For each row \(i\), we treat the \(n\) control points \(P_{i,0}, \dots, P_{i,n-1}\) as a 1D Bézier curve and apply the standard de Casteljau algorithm at parameter \(u\). This produces \(n\) intermediate points—one from each row.
  2. Fix v (column direction): We then treat these \(n\) intermediate points as control points of another 1D Bézier curve and apply de Casteljau again at parameter \(v\). The result is a single point on the Bézier surface at \((u, v)\).

In my implementation:

  • evaluateStep performs one interpolation step between adjacent 3D control points.
  • evaluate1D repeatedly calls evaluateStep until only one point remains, fully evaluating a 1D Bézier curve at parameter \(t\).
  • evaluate(u, v) first calls evaluate1D on each row using \(u\), collects the resulting intermediate points, and then calls evaluate1D again on those points using \(v\) to obtain the final surface point.

This separable approach directly generalizes the 1D de Casteljau algorithm to 2D Bézier surfaces.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

I implemented the vertex normal as an area-weighted average of the normals of all incident faces. For a given vertex, I loop over its one-ring faces using the halfedge structure; for each non-boundary face, I compute the face normal with a cross product of two edge vectors (which is automatically scaled by twice the triangle area). I then accumulate these unnormalized face normals into a single vector and finally normalize the sum to obtain a unit-length vertex normal.

  Vector3D N(0, 0, 0);

  
  HalfedgeCIter h = halfedge();

  HalfedgeCIter h0 = h;

  do {
    FaceCIter f = h->face();
    // ignore boundary faces
    if (!f->isBoundary()) {
      // positions of the three vertices of the triangle
      Vector3D p0 = h->vertex()->position;
      Vector3D p1 = h->next()->vertex()->position;
      Vector3D p2 = h->next()->next()->vertex()->position;

      // face normal (proportional to the area of the triangle)
      Vector3D n = cross(p1 - p0, p2 - p0);

      // area-weighted: just accumulate the unnormalized face normals
      N += n;
    }

    // traverse around the vertex: h -> twin -> next
    h = h->twin()->next();
  } while (h != h0);

  // finally normalize
  if (N.norm() == 0) return N;
  return N.unit();

Part 6: Loop subdivision for mesh upsampling

I implemented Loop subdivision to increase the resolution of the mesh. The algorithm follows the standard Loop subdivision steps:

  1. Compute new positions for original vertices: For each vertex in the original mesh, I computed a new position based on its neighbors using the Loop subdivision weighting formula:
    • If the vertex has 3 neighbors (boundary), \( \beta = 3/16 \).
    • Otherwise, \( \beta = 3 / (8n) \).
    • New position = \((1 - n * \beta) * \text{original position} + \beta * \sum \text{neighbor positions}\).
  2. Compute new positions for edge vertices: For each edge, I computed the position of the new vertex that would be created by splitting that edge. The formula is: newPosition = (3.0 * (a + b) + (c + d)) / 8.0, where a and b are the endpoints of the edge, and c and d are the opposite vertices of the two adjacent triangles.
  3. Split every edge: I iterated through the original edges and split them. I made sure to only split the edges that existed in the original mesh to avoid infinite loops. I stored the pre-computed new positions in the new vertices.
  4. Flip new edges: I checked every new edge. If it connected an original vertex to a new vertex (or vice-versa), I flipped it to restore the typical triangle connectivity (or rather, to ensure that the mesh is "beautified").
  5. Update positions: Finally, I copied the computed newPosition to the actual position of the vertices.

The most difficult part was ensuring the order of operations was correct and that I didn't try to split an edge that was already split in the current iteration. I handled this by counting the number of edges in the original mesh before splitting and only iterating that many times.

Observations on Loop Subdivision:

Loop subdivision tends to smooth out sharp corners and edges. Because the subdivision rule averages vertex positions, sharp features get blunted after a few iterations. To preserve sharp edges, one typically needs to have higher vertex density along the edge. Pre-splitting edges increases the number of vertices along the edge, which can help define the edge better.
